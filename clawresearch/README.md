This directory contains some working files for some claw-driven research for this repo.

Track links and candidate sources that look worth incorporating into this repo.

Goal: maintain a lightweight, accumulative queue of useful material with short notes about why each item may matter.

Suggested source classes:
- Lean community blog posts
- Month-in-Mathlib / New-in-mathlib posts
- arXiv papers
- GitHub repositories and tools
- conference/workshop announcements and reports
- Zulip/forum threads when they are directly discoverable and technically substantial

Current bias:
- Prefer high-signal domain material for combinatorics, measure theory, probability, analysis, and adjacent areas.
- Prefer practical commentary over announcements: workflow notes, architectural writeups, implementation lessons, design tradeoffs, and formalization case studies.
- Treat public Zulip discovery via generic web search as lower-yield unless a clearly valuable thread or document surfaces.

Primary artifact:
- `links.md` — curated links with one-line descriptions and rough priority

Optional later artifacts:
- `sources.md` — stable feeds/searches to re-run
- `notes/` — deeper analysis of especially promising items

Working conclusions from initial passes:
- The Lean community blog is the highest-signal recurring source.
- `Month in Mathlib` posts are especially good for domain scouting.
- arXiv is useful when a paper is clearly substantial, but many hits are lower-value than blog commentary for skill authoring.
- Maintenance/workflow artifacts also matter, not just math content; example: changelog/search/discovery resources.
